## ⭐ 核心概念

| 概念 | 说明 |
|------|------|
| **Locale** | 语言 + 国家/地区，如 `zh_CN`、`en_US` |
| **ResourceBundle** | 键值对消息资源 |
| **Unicode** | 全球字符集；Java 源码 UTF-8 |
| **格式化** | 数字、货币、日期按 Locale |

```java
Locale china = Locale.forLanguageTag("zh-CN");
Locale us = Locale.US;

NumberFormat currency = NumberFormat.getCurrencyInstance(china);
String price = currency.format(1234.5);  // ¥1,234.50

NumberFormat percent = NumberFormat.getPercentInstance();
percent.format(0.125);  // 12%
```

## ⭐ ResourceBundle

### properties 文件

```properties
# messages_zh_CN.properties
greeting=你好，{0}！
error.not_found=资源 {0} 不存在
order.status.paid=已支付

# messages_en_US.properties
greeting=Hello, {0}!
error.not_found=Resource {0} not found
order.status.paid=Paid
```

```java
Locale locale = Locale.forLanguageTag("zh-CN");
ResourceBundle bundle = ResourceBundle.getBundle("messages", locale);
String template = bundle.getString("greeting");
String msg = MessageFormat.format(template, "Tom");
```

**UTF-8 properties（Java 9+）：** 原生 UTF-8；老项目用 `native2ascii` 或 `.properties` 转义。

### ListResourceBundle / Class 方式

```java
public class Messages_zh_CN extends ListResourceBundle {
  @Override
  protected Object[][] getContents() {
    return new Object[][] {
        { "greeting", "你好" }
    };
  }
}
```

大型项目更常用 properties + 构建时校验缺失 key。

## ⭐ 日期与数字格式化

```java
// 日期 — 优先 java.time + DateTimeFormatter
DateTimeFormatter fmt = DateTimeFormatter
    .ofLocalizedDate(FormatStyle.LONG)
    .withLocale(Locale.CHINA);
LocalDate.now().format(fmt);

// 排序比较
Collator collator = Collator.getInstance(Locale.CHINA);
collator.compare("张三", "李四");
```

## ⭐ Spring 中的 i18n

```java
@Configuration
public class I18nConfig {
  @Bean
  public MessageSource messageSource() {
    ResourceBundleMessageSource ms = new ResourceBundleMessageSource();
    ms.setBasenames("messages");
    ms.setDefaultEncoding("UTF-8");
    return ms;
  }
}

@Service
public class OrderService {
  @Autowired MessageSource messageSource;

  public String statusLabel(OrderStatus status, Locale locale) {
    return messageSource.getMessage(
        "order.status." + status.name().toLowerCase(),
        null, locale);
  }
}
```

### REST 多语言

```java
@GetMapping("/orders/{id}")
public OrderDto get(@PathVariable Long id,
    @RequestHeader(value = "Accept-Language", defaultValue = "zh-CN") Locale locale) {
  Order order = service.find(id);
  return mapper.toDto(order, locale);
}
```

或 URL 前缀：`/en/api/...`、`/zh/api/...`。

## 📌 数据库与 i18n

| 策略 | 说明 |
|------|------|
| _code 存库 + 前端/后端翻译 | 常见 |
| 多语言列 `name_zh`, `name_en` | 简单 CMS |
| 独立翻译表 | 复杂产品 |

**不要** 把翻译后的文案当主键；用 stable code。

## 📌 国内企业 vs 出海

| 场景 | 做法 |
|------|------|
| 纯国内系统 | 固定中文，error message 中文 |
| 出海 SaaS | MessageSource + Accept-Language |
| 报表/Excel 导出 | 按用户 Locale 格式化日期金额 |

---

## ⚠️ 常见坑

| 坑 | 说明 |
|----|------|
| 硬编码中文抛异常 | 无法国际化 |
| 拼接复数 | 用 MessageFormat 或 ICU4J |
| 时区与 Locale 混淆 | Locale 管格式，ZoneId 管时刻 |
| 缺失 key | 启动时扫描或测试覆盖 |

---

## 本章小结

- Locale + ResourceBundle + MessageFormat 是 JDK 基础
- Spring `MessageSource` 整合 REST 多语言
- 国内项目可略读；出海必系统做 i18n
