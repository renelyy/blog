# JVM 内存结构

[← 返回索引](./index)

> **本节目标：** 理解运行时数据区、对象布局、类加载与 Java 内存模型（JMM）基础。

---

## ⭐ JVM 整体结构

```text
┌─────────────────────────────────────────┐
│              JVM 运行时                    │
├──────────────┬──────────────────────────┤
│  线程私有     │  堆 Heap（对象、数组）      │
│  · 程序计数器  │  方法区 / 元空间 Metaspace │
│  · 虚拟机栈    │  （类元数据、常量池等）      │
│  · 本地方法栈  │  直接内存 Direct Memory    │
└──────────────┴──────────────────────────┘
```

| 区域 | 线程 | 说明 |
|------|------|------|
| **程序计数器** | 私有 | 当前字节码行号 |
| **虚拟机栈** | 私有 | 栈帧：局部变量表、操作数栈、动态链接、方法出口 |
| **本地方法栈** | 私有 | Native 方法 |
| **堆** | 共享 | **对象实例**、数组 — GC 主战场 |
| **元空间** | 共享 | JDK 8+ 类元数据（替代永久代 PermGen） |
| **直接内存** | — | NIO DirectByteBuffer，不在堆内 |

## ⭐ 堆的分代

```text
                    Java Heap
    ┌─────────────────────────────────────┐
    │  年轻代 Young                          │
    │  ┌─────────┬──────────┬──────────┐  │
    │  │  Eden   │ Survivor0│ Survivor1│  │
    │  └─────────┴──────────┴──────────┘  │
    ├─────────────────────────────────────┤
    │  老年代 Old Generation               │
    └─────────────────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| **Eden** | 新对象分配 |
| **Survivor** | Minor GC 后存活对象复制 |
| **Old** | 长期存活对象；Full GC 扫描 |

**大对象** 可能直接进入老年代（`-XX:PretenureSizeThreshold`）。

## ⭐ 栈帧与方法调用

```java
public int add(int a, int b) {
  return a + b;
}
```

```text
栈帧 add:
  局部变量表: a=1, b=2
  操作数栈:   [1][2] → iadd → [3]
  返回        ireturn
```

| 概念 | 说明 |
|------|------|
| **StackOverflowError** | 递归过深、栈帧过多 — 默认栈大小 `-Xss` |
| **OutOfMemoryError: unable to create native thread** | 线程过多 — 栈 × 线程数 |

## ⭐ 对象内存布局（HotSpot）

```text
对象头 Header
  ├── Mark Word（哈希、锁状态、GC 年龄）
  └── Klass Pointer（类元数据指针）
实例数据 Instance Data
对齐填充 Padding（8 字节对齐）
```

| 类型 | 大小（64 位，压缩指针开启） |
|------|---------------------------|
| 空 Object | 约 12～16 字节 |
| `new Integer(1)` | 头 + int 字段 + 对齐 |

**指针压缩：** `-XX:+UseCompressedOops` — 堆 < 32GB 时引用 4 字节，省内存。

## ⭐ 对象分配与逃逸分析

| 路径 | 说明 |
|------|------|
| **Eden 分配** | 默认 `new` |
| **TLAB** | 线程本地分配缓冲，减少同步 |
| **栈上分配** | 逃逸分析后标量替换 — 对象未逃出方法 |
| **堆外** | DirectByteBuffer、Unsafe |

**逃逸：** 对象被返回、存入堆字段、跨线程可见 → 必须在堆上。

---

## ⭐ 类加载与元空间

```text
加载 → 验证 → 准备 → 解析 → 初始化
```

| 阶段 | 说明 |
|------|------|
| **加载** | 读 class，生成 Class 对象 |
| **准备** | 静态变量零值 |
| **初始化** | `<clinit>` 执行静态块 |

**双亲委派：** 先让父加载器加载，保证核心类安全。

**元空间 OOM：** 类加载过多（动态代理、Groovy、热部署）→ `-XX:MaxMetaspaceSize`。

JDK 8 前 **PermGen** 固定上限；JDK 8+ **Metaspace** 用本地内存，默认仅受系统内存限制。

---

## ⭐ Java 内存模型 JMM（与 JVM 内存区分）

| 概念 | 说明 |
|------|------|
| **主内存** | 共享变量所在堆 |
| **工作内存** | 线程本地缓存副本 |
| **happens-before** | 可见性规则 |

与 [并发编程](../java/core-java/vol1/12-concurrency) 中 `volatile`、`synchronized` 直接相关 — **JMM 是规范，JVM 内存结构是实现载体**。

## 📌 企业容器部署注意

| 参数 | 说明 |
|------|------|
| `-Xms` / `-Xmx` | 堆初始/最大 — **建议相等**，避免动态扩缩 |
| 容器 memory limit | K8s `limits.memory` 要大于 **堆 + 元空间 + 栈 + 直接内存 + 本地库** |
| **不要** 把 `-Xmx` 设成容器 100% | 留 20%～30% 给非堆 |

```bash
# 容器 2Gi 示例
JAVA_OPTS="-Xms512m -Xmx512m -XX:MaxMetaspaceSize=256m"
```

见 [Boot 生产运维](../spring/boot/11-production)。

## ⚠️ 常见坑

| 坑 | 说明 |
|----|------|
| 把 JVM 内存当只有堆 | 直接内存、元空间也会 OOM |
| `-Xmx` 等于容器 limit | 被 OOMKilled |
| 静态集合无限增长 | 老年代满 — **逻辑泄漏** |
| ThreadLocal 未 remove | 线程池复用导致泄漏 |
| 混淆 JMM 与 JVM 堆栈 | 面试与排查两套概念 |

---

## 本章小结

- **堆** 放对象，**栈** 放方法帧，**元空间** 放类元数据
- 年轻代/老年代 — 为 GC 分代服务
- 容器环境堆与非堆要一起算；JMM 管并发可见性
